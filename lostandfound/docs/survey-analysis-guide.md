# 调查问卷数据分析指南

## 一、问卷设计说明

### 1.1 问卷结构

本问卷共分为6个部分，28道题目：

| 部分 | 题目数量 | 主要内容 | 分析目的 |
|------|---------|---------|---------|
| 第一部分 | 4题 | 基本信息 | 了解受访者背景，进行分组分析 |
| 第二部分 | 4题 | 现有方式评价 | 识别现有问题，论证研究必要性 |
| 第三部分 | 8题 | 功能需求 | 验证系统功能设计的合理性 |
| 第四部分 | 4题 | 体验与安全 | 优化用户体验和安全设计 |
| 第五部分 | 5题 | 使用意愿 | 评估系统可行性和推广价值 |
| 第六部分 | 3题 | 开放性问题 | 收集深度反馈和改进建议 |

### 1.2 题型分布

- **单选题**：18题（用于明确态度和偏好）
- **多选题**：7题（用于全面了解需求）
- **开放题**：3题（用于收集深度意见）

---

## 二、数据收集建议

### 2.1 样本量要求

- **最小样本量**：100份有效问卷
- **推荐样本量**：150-200份
- **样本构成**：
  - 本科生：60-70%
  - 研究生：20-25%
  - 教职工：10-15%

### 2.2 发放渠道

1. **线上渠道**（主要）
   - 校园论坛/贴吧
   - 班级/年级微信群/QQ群
   - 学生会/社团组织
   - 校园公众号

2. **线下渠道**（辅助）
   - 图书馆、食堂等人流密集区
   - 宿舍楼发放
   - 课堂调研

### 2.3 质量控制

- 设置必答题，确保关键数据完整
- 设置逻辑跳转，提高问卷效率
- 限制IP重复提交
- 设置答题时间下限（建议≥2分钟）
- 人工审核开放题答案质量

## 三、数据分析方法

### 3.1 描述性统计分析

#### 3.1.1 基本信息分析

**分析指标：**
- 受访者身份分布（Q1）
- 性别分布（Q2）
- 丢失物品经历（Q3）
- 捡到物品经历（Q4）

**分析方法：**
- 频数统计
- 百分比计算
- 饼图/柱状图展示

**预期结果示例：**
```
受访者身份分布：
- 本科生：65%
- 研究生：23%
- 教职工：12%

丢失物品经历：
- 经常（3次以上）：15%
- 偶尔（1-2次）：58%
- 从未丢失过：27%
```

#### 3.1.2 现有方式评价分析

**关键题目：Q5-Q8**

**分析重点：**
1. **现有方式使用情况**（Q5）
   - 统计各渠道使用频率
   - 识别主流渠道

2. **满意度分析**（Q6）
   - 计算满意度均值
   - 分析不满意原因

3. **问题识别**（Q7）
   - 统计各问题提及率
   - 排序前5大痛点

4. **成功率评估**（Q8）
   - 计算平均找回率
   - 论证改进必要性

**论文写作要点：**
- 用数据说明现有方式的不足
- 为系统开发提供需求依据

#### 3.1.3 功能需求分析

**关键题目：Q9-Q16**

**分析方法：**

1. **使用意愿分析**（Q9）
   ```
   使用意愿 = (一定会使用×100% + 可能会使用×75%) / 总人数
   ```
   - 目标：使用意愿≥70%

2. **功能重要性排序**（Q10-Q15）
   - 计算各功能"非常需要"+"比较需要"的比例
   - 排序功能优先级
   - 绘制需求热力图

3. **AI功能接受度**（Q11）
   - 统计需求程度分布
   - 分析不同群体差异

4. **LBS功能接受度**（Q12）
   - 评估地图功能价值
   - 分析隐私顾虑

5. **智能匹配接受度**（Q13-Q14）
   - 验证核心功能合理性
   - 确定匹配算法权重

6. **积分激励接受度**（Q15-Q16）
   - 评估激励机制有效性
   - 优化积分规则设计

**预期结果示例：**
```
功能需求排序（需求度≥80%）：
1. 智能匹配推荐：92%
2. 消息通知提醒：88%
3. 地图定位功能：85%
4. AI图像识别：82%
5. 积分激励体系：76%
```

### 3.2 交叉分析

#### 3.2.1 身份与需求关系

**分析维度：**
- 不同身份群体对各功能的需求差异
- 不同丢失经历者的功能偏好

**分析方法：**
- 卡方检验（Chi-square test）
- 列联表分析

**示例分析：**
```
身份 × AI功能需求（Q1 × Q11）
- 本科生需求度：85%
- 研究生需求度：78%
- 教职工需求度：65%
→ 结论：年轻用户对AI功能接受度更高
```

#### 3.2.2 满意度与使用意愿关系

**分析维度：**
- 现有方式满意度与新平台使用意愿的关系（Q6 × Q9）

**分析方法：**
- 相关性分析
- 回归分析

**预期结论：**
- 对现有方式不满意的用户，使用新平台意愿更强

### 3.3 开放题分析

**题目：Q26-Q28**

**分析方法：**
1. **内容分析法**
   - 提取高频关键词
   - 归纳主题类别

2. **情感分析**
   - 识别正面/负面/中性情绪
   - 统计情感倾向分布

3. **需求挖掘**
   - 提取未覆盖的功能需求
   - 识别潜在改进点

**工具推荐：**
- 词云生成：WordCloud
- 文本分析：Python NLTK / jieba分词
- 人工编码分类

## 五、数据处理工具

### 5.1 推荐工具

| 工具 | 用途 | 优势 |

| 问卷星/腾讯问卷 | 在线收集、自动统计 | 便捷高效，实时查看 |

### 5.2 Python分析示例

```python
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

# 读取数据
df = pd.read_excel('survey_data.xlsx')

# 基本统计
print(df['Q1_身份'].value_counts())
print(df['Q6_满意度'].value_counts(normalize=True))

# 交叉分析
cross_tab = pd.crosstab(df['Q1_身份'], df['Q11_AI需求'])
print(cross_tab)

# 可视化
plt.figure(figsize=(10, 6))
df['Q6_满意度'].value_counts().plot(kind='bar')
plt.title('现有方式满意度分布')
plt.xlabel('满意度')
plt.ylabel('人数')
plt.savefig('satisfaction.png')
plt.show()

# 需求度计算
def calculate_demand(series):
    demand = (series.isin(['非常需要', '比较需要']).sum() / len(series)) * 100
    return round(demand, 1)

features = ['Q11_AI需求', 'Q12_LBS需求', 'Q13_匹配需求', 'Q15_积分需求']
demand_scores = {f: calculate_demand(df[f]) for f in features}
print('功能需求度：', demand_scores)
```

## 六、常见问题与解决方案

### 6.1 样本量不足

**问题**：有效问卷数量少于100份

**解决方案**：
1. 扩大发放渠道
2. 提供小额激励（如抽奖��
3. 寻求老师/辅导员协助
4. 延长收集时间

### 6.2 数据质量差

**问题**：大量无效问卷（答题时间过短、答案明显随意）

**解决方案**：
1. 设置答题时间下限
2. 增加注意力检查题
3. 人工筛选无效问卷
4. 优化问卷设计，减少题目数量

### 6.3 开放题回答率低

**问题**：开放题空白率超过50%

**解决方案**：
1. 将开放题设为选填
2. 提供示例引导
3. 减少开放题数量
4. 调整开放题位置

### 6.4 结果不符合预期

**问题**：某些功能需求度低于预期

**解决方案**：
1. 分析原因（功能描述不清、受访者理解偏差等）
2. 进行补充访谈
3. 调整系统设计，突出高需求功能
4. 在论文中客观呈现，分析原因
